恰巧我也喜欢跟风尝鲜用过 hexo 和 github 以及其他写作平台

  1. 给我的一个印象是 metaspace 把 hexo 和 github 给包装了一层，方便了用户创建自己的博客，省略了部署和编辑、发布的麻烦，确实减轻了非程序猿用户的门槛。不过如果 github 把我的仓库删了怎么办？也许我一不小心骂了微软一句，这种事谁知道呢？AR 里存的用户的文章信息还可以恢复出来？估计可以但也会很麻烦，所以团队说的删帖这种事，那就看 github 政策了。（这段只是疑问，不算吐槽哈）
  2. 作者说请大家忍受下 web3 substack 一系列反人类的 UX，又逗我，哥们，你知道反人类还不改改？所见即所得，用起来图个爽，用户不都这样嘛！我写了一篇文章，在后台找不到浏览入口，左边有 create、publish、草稿，就是没有查看文章这样的按钮，放到刚发布的连接上，连个“小手手”都没有，我还是把域名 copy 放到其他浏览器上才能看到我方才写了啥。写完了咋看？这个小动作挺困扰人的，一个小玩意就会拦住很多用户进行下一步的探索欲望
  3. 终于打开文章后，在 mac 上看到的效果图跟手机屏幕一样，我放两张图，一张 metasapce 给出来的效果，一张是我自己前几年部署在 github 上的 hexo 文章图，感受下！当然这可以调整，但这些都是用户使用时的第一观感，因为外边有很多 web3 甚至 mirror 的产品，用户自然会进行对比。。

  1. 用户进入 metanetwork 首页的时候让选择 space 空间，我估计就是 storage 和 domain 等信息，用户进入首页后，提示让选择 space，估计应为我是从 console 直接进入的，已经设置好了 storage 等相关信息，所以首页的位置总是提示 space 选择失败，另外也建议团队在用户选择 space 成功后，再次进入首页时，给个“提示”，所见即所得，让用户看见哦擦，这是我的 space，酷！现在点击左侧的“六边形”小按钮啥反应没有。
  2. 另外用户用 metamask 登陆也不顺畅，总有小错，还是建议团队开放测试，邀请更多普通用户进行反馈，先满足这些基本需求再开发其他先进的功能。倒不用比 web2 博客产品号多少，基本用户体感保持一致先满足。很多团队在做 web3 应用产品的时候，其实用户会拿原有的 web2 产品进行对比的，又不像 metamask 和 cex、dex 产品一样，以前没这些东西，用户一看比较新，忍一忍也 ok，但是这些新开发的应用类产品就没有这个优势，还是得把产品体验提高提高。
  3. 当然 metanetwork 有个好的地方，即 publish 文章时比较顺畅，不用每次都调用 metamask，其实一直调用第三方钱包有点烦人，前几天用 lens 就是，每次发条信息就要调用一次 metamask，挺烦人的。
  4. 还有不少吐槽的地方，团队也在逐步完善，以后慢慢来吧，毕竟自己还是 crypto 领域的韭菜，对本行业的很多门槛都已经免疫了，如果你给传统用户去用这个估计会被。。。。

总结：Web3 的同志们 加油 加油，未来是我们的。
